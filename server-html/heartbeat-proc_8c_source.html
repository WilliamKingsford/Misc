00009 /*
00010  * Block sizes update algorithm:
00011  *
00012  * 1. When heartbeat starts, it send the list of all current blocks to monitor;
00013  * 2. When a new block is sent to this chunk server, heartbeat will receive
00014  *    a signal from block manager. It will add the block metadata into
00015  *    priv->block_metadata_new queue. If more than 30 block metadata have been
00016  *    queued, heartbeat proc will send the list to monitor.
00017  * 3. On an idle chunk server, it may take long time to queue 30 blocks.
00018  *    To keep the monitor up-to-date, a flush timer is scheduled to flush
00019  *    block info to monitor every 30 seconds.
00020  */
