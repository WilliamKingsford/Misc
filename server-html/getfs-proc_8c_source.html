00023 /*
00024  * Implementation Notes:
00025  *
00026  * Checking and writing of fs objects are completely asynchronous in this processor.
00027  * - FS object checking is done by a worker thread.
00028  * - Writing of received fs objects is done with the async obj-store API.
00029  *
00030  * At the beginning, all root object id is put into inspect queue. And then
00031  * We start a worker thread to check the first object in the inspect queue.
00032  *
00033  * After the worker thread is done, we send object requests in the main thread.
00034  * And then we start a worker to check the next object in the inspect queue.
00035  *
00036  * After an object is received and store asynchronously into disk, and if the object
00037  * is a directory, we put it into the inspect queue. And then we start a worker
00038  * to check the next object in the inspect queue, if no worker is running.
00039  * This means only 1 worker can be running at the same time. Because we use thread
00040  * pool, there will be no performance problem of creating threads.
00041  *
00042  * The end condition is checked after:
00043  * - worker thread is done
00044  * - an object is written
00045  * The end condition is
00046  * - inspect queue is empty, and
00047  * - no object request is pending, and
00048  * - no worker is running
00049  */
